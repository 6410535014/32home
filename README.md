## 📌 Overview
โดยปกติลูกบ้านจะชำระค่าส่วนกลางเป็นรายปี แต่ปัญหาคือหลังจากจ่ายเงินไปแล้ว ลูกบ้านมักไม่ทราบความเคลื่อนไหวทางการเงินของนิติบุคคล จนกว่าจะมีการประชุมใหญ่ปลายปี รวมถึงการจัดซื้อทรัพย์สินเข้าหมู่บ้านที่หากมีการจัดซื้อโดยไม่สอบถามความต้องการของลูกบ้านเลย ก็อาจทำให้เกิดความสงสัยในเรื่องความโปร่งใสของการใช้เงินได้

**32Home** จึงถูกออกแบบมาเพื่อแก้ไขปัญหานี้ โดยเน้น **"ความโปร่งใสแบบส่งถึงบ้าน"** คือจะไม่รอจนกว่าจะมีการประชุมใหญ่ปลายปี แต่จะทำให้ลูกบ้านสามารถดูสรุปรายจ่ายประจำเดือนได้เลยเมื่อต้องการ โดยจะแจกแจงรายละเอียดการใช้จ่ายจริงเป็นตัวเงินอย่างละเอียด ควบคู่ไปกับระบบ E-Voting ที่โปร่งใส เพื่อให้ลูกบ้านมั่นใจว่าเงินที่จ่ายเป็นค่าส่วนกลางจะถูกบริหารจัดการอย่างโปร่งใส

## 🚀 Use Case / Scenario

### สถานการณ์ที่ 1: การแจ้งยอดการใช้จ่ายรายเดือน
คุณเอจ่ายค่าส่วนกลางรายปีไปแล้วเมื่อเดือนมกราคม จำนวน xxxxxx บาท

วันที่ 5 ของเดือนพฤษภาคม ระบบแจ้งเตือนทางแอป หัวข้อ "รายงานสรุปค่าใช้จ่ายหมู่บ้าน ประจำเดือนเมษายน"

คุณเอเปิดดู ไม่ใช่กราฟวงกลมรวม ๆ แต่เห็นตารางแจกแจงรายการจริง เช่น
* ค่าจ้าง รปภ. xxxxxx บาท
* ค่าไฟฟ้าส่วนกลาง xxxxxx บาท
* รวมรายจ่ายทั้งหมู่บ้านเดือนนี้ xxxxxx บาท
* สถานะเงินกองทุนคงเหลือ xxxxxx บาท

คุณเอรู้สึกสบายใจที่เห็นตัวเลขละเอียดทุกบาททุกสตางค์ และรู้สถานะการเงินของหมู่บ้านโดยไม่ต้องรอรอบประชุมปีหน้า

### สถานการณ์ที่ 2: การโหวตจัดซื้อพร้อมดูสถานะการเงิน
นิติบุคคลเสนอวาระจัดซื้อเครื่องออกกำลังกายใหม่ 3 เครื่อง งบ xxxxxx บาท

ก่อนโหวต คุณบีกดดู "งบประมาณคงเหลือหมวดสันทนาการ" ในแอป เห็นตัวเลขชัดเจนว่ามีงบส่วนนี้อยู่ xxxxxx บาท ซึ่งมีมากกว่างบที่ต้องใช้สำหรับซื้อเครื่องออกกำลังกาย

คุณบีกดโหวต "อนุมัติ" ผ่านระบบ E-Voting
ผลโหวตถูกบันทึกและล็อคผลทันทีแก้ไขไม่ได้

เดือนถัดมา ในรายงานรายจ่ายประจำเดือน จะมีรายการ "ค่าเครื่องออกกำลังกาย xxxxxx บาท" ปรากฏขึ้น เพื่อยืนยันว่าเงินถูกใช้ไปตรงตามมติที่โหวตจริง ๆ

## ✨ Features

### 1. ระบบรายงานรายจ่ายรายเดือน
* **รายการสินค้าโดยละเอียด:** แสดงรายการค่าใช้จ่ายแยกเป็นรายหมวดหมู่และรายชิ้น
* **การแสดงจำนวนเงินที่แน่นอน:** แสดงตัวเลขจำนวนเงินจริงของแต่ละรายการ ไม่ใช่แค่เปอร์เซ็นต์ เพื่อความชัดเจนสูงสุด
* **เอกสารแนบหลักฐาน:** รายการใหญ่ ๆ สามารถกดเพื่อดูภาพถ่ายหน้างาน หรือรูปใบเสร็จรับเงินจากผู้รับเหมาได้ทันที

### 2. ระบบโหวต
* **การบันทึกการลงคะแนนที่ไม่สามารถเปลี่ยนแปลงได้:** คะแนนเสียงทุกคะแนนจะถูกบันทึก และไม่สามารถแก้ไขผลโหวตย้อนหลัง
* **การโหวตโดยเชื่อมโยงกับสถานะการเงิน:** การโหวตเรื่องงบประมาณจะเชื่อมโยงกับบัญชีจริง หากงบประมาณไม่เพียงพอ ระบบจะแจ้งเตือนทันทีว่าไม่สามารถเปิดโหวตวาระนี้ได้

### 3. แดชบอร์ดสุขภาพการเงิน
* **สถานะการเงิน:** แสดงยอดเงินคงเหลือในบัญชีนิติบุคคลปัจจุบัน เพื่อให้ลูกบ้านทราบเสถียรภาพทางการเงินของหมู่บ้าน

## 🛠 Technology

### 1. ระบบซิงค์ข้อมูลบัญชีผ่าน CLI
การอัปเดตข้อมูลยอดเงินและรายการค่าใช้จ่ายผ่านสคริปต์ Node.js ที่รองรับการอ่านไฟล์ CSV/Excel โดยตรง

### 2. ฐานข้อมูลคลาวด์แบบเรียลไทม์
ใช้ Google Firestore เพื่อให้ลูกบ้านทุกคนมองเห็นยอดเงินส่วนกลางและรายการค่าใช้จ่ายชุดเดียวกัน ณ ปัจจุบัน ทันทีที่อัปเดตข้อมูล

### 3. ระบบลงคะแนนดิจิทัลอัตโนมัติ
ระบบจัดการวาระการโหวตที่กำหนดเวลาเปิด-ปิดได้ล่วงหน้า พร้อมระบบประมวลผลคะแนนและแสดงสัดส่วนในรูปแบบกราฟ  ทันทีที่การโหวตสิ้นสุดลง
